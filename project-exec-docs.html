Executable Docs: Making Developer Documentation Interactive and Intuitive

Context & Challenge

At Microsoft Azure, one of the key factors that influence the adoption of cloud services is the ease and speed of deployment. Customers who can quickly set up and run applications or Proofs-of Concept on the cloud are more likely to convert. Therefore, it is important to support quick, successful deployments for new users to Azure, especially for connected resources that require complex configuration and integration.

The challenge was to design a seamless, intuitive experience that integrates live code execution without overwhelming users or disrupting their workflow.

My Role

As the UX Researcher on this project, I partnered closely with product managers, engineers, and designers to deeply understand how developers consume documentation and interact with Linux deployment stacks. My responsibility was to uncover user needs, identify pain points, and validate design decisions through rigorous research.

Research Approach

To capture a comprehensive picture of developer behavior and needs, I employed a multi-method research strategy:
  • Semi-Structured Interviews: Conducted in-depth interviews with Azure, AWS, and GCP developers and DevOps engineers to explore current learning and onboarding habits and frustrations.
  • Usability Testing: Facilitated remote sessions where participants used early designs and interactive prototypes of Executable Docs, focusing on code execution, script personalization, and navigation of steps.
  • Analytics: Gathered quantitative data on feature usage and conversion to complement qualitative insights.

Key Insights

Participants overwhelmingly preferred guided deployment during the study. Participants saw it as a way to sanity check their deployments, and wanted to understand exactly what the script was going to do.

The visibility of the scripts, guided explanations, and table of contents overview were perceived as a positive change from the current portal and command line experiences.

Participants strongly expressed the need for more customization of the enviornment variables and scripts, particularly to mitigate concerns over cost and security.

Solutions & Impact

Based on these insights, we iterated on the design to:
  • Implement a guided deployment experience with clear explanations of each step and the ability to execute code snippets directly within the documentation.
  • Provide a customizable script editor that allows users to modify environment variables and scripts before execution.
  • Enhance the navigation experience with a table of contents overview and dynamic updating of current status, enabling users to quickly access relevant sections and code snippets.
  • Integrate dynamic visual indicators for code execution states (running, success, failure).

Post-launch metrics showed a significant increase in Linux deployments and user engagement with executable docs. Feedback from follow up usability studies highlighted improved confidence and efficiency when using Executable Docs.

Overall, the Executable Docs project successfully transformed developer documentation into an interactive, user-friendly experience that empowers users to understand and deploy applications quickly and confidently on Azure.
